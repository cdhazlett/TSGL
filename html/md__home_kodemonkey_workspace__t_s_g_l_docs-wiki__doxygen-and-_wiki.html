Doxygen-and-Wiki

***NOTE:*** This process is for people who have cloned the repository on Linux/Mac. Windows users may use Git Bash, though that has not been tested.

***NOTE:*** Before updating the Doxygen, please see the READMEMISC.txt file. You will not be able to update the Doxygen correctly if you fail to read READMEMISC.txt first!

***BIG NOTE:*** You ***NEED*** doxygen 1.8.6 in order to update the documentation correctly. Previous versions will cause a ***LOT*** of problems.

In order to provide an API and extensive documentation for the TSGL library, we rely on a tool known as Doxygen.

We also use the Wiki pages to provide important information about the TSGL library (Debian and RPM packages, FAQs, etc...).

If you need to update our Doxygen pages (you've added a new class, found a typo and fixed it, etc...), or need to update the Wiki, follow this process (via terminal or command-line):

1). Navigate into the TSGL root directory (where the Makefile is).

2). ***IF*** you are updating the Wiki pages, make sure any changes are saved and made ***BEFORE*** continuing.

3). Type "git submodule update --remote". This should cause the docs-wiki/ submodule to update (and docs/ if it has been updated recently).

4). Type "git add ." . This should add the docs-wiki/ submodule (and possibly the docs/ submodule if it has been updated recently).

5). Do a "git commit -m "" ".

6). Type "git push origin master" . This should push the changes to the main GitHub branch.

***IF*** you are only updating the Wiki pages, and not the Doxygen pages, you should stop here.

Otherwise, please continue on to the next steps.

7). Type "make cleandocs".

8). cd into the docs/ directory.

9). Type "rm -r html".

10). cd back out of the docs/ directory (cd ../).

11). Type "make docs".

12). cd into the docs/ directory again.

13). Type "git add --all" .

14). Do a "git commit -m "" " inside of the docs/ directory.

15). Type "git push origin gh-pages" . This should push the new Doxygen pages to our gh-pages branch (where our documentation ***SHOULD*** reside).

16). cd back out of the docs/ directory (cd ../).

17). Type "git add ." .

18). Do a "git commit -m "" ".

19). Type "git push origin master". This should update the docs/ submodule inside of our main GitHub branch.

That's it!
